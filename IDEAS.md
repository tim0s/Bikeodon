# Bikeodon feature ideas

## Quick wins
- **Re-post button** in the activity detail page — useful if you changed settings and want a fresh post
- **Manual sync button** in the web UI so you don't have to wait 15 minutes
- **Activity filtering** in the list — by sport type, date range, or posted/unposted
- **Preview post** before it goes out — show the rendered text and images in the UI

## Moderate effort
- **Garmin Connect support** — second activity source alongside Strava; many cyclists use both
- **Custom post scheduling** — post at a specific time rather than immediately on sync
- **Multiple Mastodon accounts** — post to different instances, or cross-post
- **Dark map tiles** — already supported via tile URL setting, but add a quick-switch toggle in the UI
- **Webhook / push** instead of polling — Strava supports webhooks, which would mean instant sync instead of 15-minute delay

## Bigger features
- **Activity statistics page** — weekly/monthly distance totals, elevation charts, streak tracking
- **Bluesky support** — post to AT Protocol in addition to or instead of Mastodon
- **Segment highlights** — detect and annotate Strava segments or PRs in the post text
- **Elevation profile overlay** — add a small elevation chart burned into the map image itself
- **Multi-user public instance** — proper signup flow, admin panel, rate limiting; turn it into a hosted service others can sign up for

## Interesting / niche
- **Year in review image** — auto-generate an annual summary post (total km, elevation, activity count)
- **Route comparison** — overlay multiple rides on the same map to see repeated routes
- **Heatmap mode** — aggregate all activities into a single heatmap image for profile/banner use
